# 当前环境背景

患者端已经具备了预约挂号和查询我的记录的能力。目前前端有两个明显的待开发预留位：

# 任务目标：开发 AI 智能导诊聊天室，并打通预约记录的取消（删除）功能

请在 `backend` 和 `frontend` 目录下协同修改/创建以下文件：

## 第一步：后端增加“AI智能导诊”流式接口

请在 `backend/api/endpoints/chat.py` 页面（或相关路由文件中）增加一个专属端点：

- **路由**：`POST /api/chat/triage`

- **请求体**：接收 `{"message": "用户输入的症状"}`。

- **业务逻辑**：不需要像医生工作台那样去查复杂的医学指南库或病历，直接初始化一个轻量的大模型实例。为其注入一个**导诊提示词（System Prompt）**：

  > "你是一位专业的医院导诊台护士。请根据患者描述的症状，用亲切、通俗的语言分析他们可能存在健康问题，并明确推荐他们去哪一个具体的科室挂号（可供选择的科室包含：消化内科、心内科、内科、外科、儿科、妇产科、急诊科、骨科、眼科、皮肤科）。最后给出一句温馨的就医提示。"

- **返回格式**：使用 `StreamingResponse` 返回大模型吐字的 SSE 流式文本。

## 第二步：后端增加“取消/删除预约”接口

请在 `backend/api/endpoints/appointment.py` 中增加删除或变更状态的接口：

- **路由**：`DELETE /api/appointments/{appointment_id}`
- **鉴权**：注入 `current_user: User = Depends(get_current_user)` 依赖。
- **数据库逻辑**：
  - 使用 `AsyncSession` 查询该 `appointment_id` 的预约记录。
  - 安全校验：必须确保该记录的 `patient_id == current_user.id`，防止越权操作！
  - 执行操作：可以直接从数据库执行 `db.delete(appointment)`（物理删除），或者将它的 `status` 更新为 `'已取消'`（逻辑删除，此处建议直接使用物理删除或更符合业务的逻辑，请你根据表结构选择最佳实现）。
  - 执行 `await db.commit()`，成功后返回 `{"message": "取消预约成功"}`。

## 第三步：前端重构“AI 智能导诊”页面

请彻底重构 `src/views/patient/Chatbot.vue`，将其打造成一个科技感十足的对话聊天室：

1. **界面布局**：
   - 上方：滚动区域，用气泡样式区分“用户发送的消息”（靠右，绿色或浅蓝背景）和“AI导诊回复的消息”（靠左，白色或淡灰色背景，支持 markdown 渲染）。
   - 底部：一个固定的输入框区域，包含文本域和“发送”按钮。
2. **流式通信**：
   - 点击发送时，将用户消息推入 `messages` 数组。
   - 使用原生 `fetch` 请求后端的 `/api/chat/triage` 接口，使用读取器 (`response.body.getReader()`) 逐字获取数据，实时累加展示在界面上，并保持聊天区域自动滚动探底。
3. **欢迎语**：在 `onMounted` 时，默认给一条导诊欢迎语：“您好！我是您的智能导诊助手，请告诉我您身体目前有哪些不舒服的症状（例如：头晕、肚子右下角隐隐作痛等），我来为您推荐合适的科室。”

## 第四步：前端“我的预约记录”页面增加“取消预约”动作

请修改 `src/views/patient/Records.vue` 页面：

1. **表格栏扩充**：在 `<el-table>` 的最右侧增加一个“操作”列。
2. **条件渲染按钮**：
   - 在操作列里放一个“取消预约”的按钮 (`<el-button type="danger" link>`)。
   - **核心交互条件**：只有当该行的状态 `status === '待就诊'` 时，才显示该取消按钮；如果状态已经是“已完成”或“已取消”，则不显示或禁用。
3. **防误触确认**：包裹一个 Element Plus 的 `<el-popconfirm>`（气泡确认框），提示文字：“确认要取消本次挂号预约吗？”
4. **触发动作**：用户点击确认后，调用封装好的 `request.delete(\`/api/appointments/${row.id}`)`。
5. **刷新状态**：请求成功后，弹出 `ElMessage.success('已成功取消预约')`，并**重新触发获取列表的方法**，让表格数据实现无缝无刷新的更新。
